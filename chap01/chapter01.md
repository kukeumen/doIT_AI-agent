# Chapter 01. LLM의 한계를 보완하는 기술

## 학습 목표

LLM(Large Language Model)은 자연스러운 텍스트 생성 능력을 갖추고 있지만, 실제 서비스를 개발하기 위해서는 여러 한계가 존재한다.

이번 장에서는 LLM의 대표적인 한계와 이를 보완하기 위한 기술들을 정리하였다.

---

## LLM의 한계

### 1. 최신 정보를 알 수 없다.

LLM은 학습 시점 이후의 정보를 알지 못한다.

예시)

* 오늘 삼성전자 주가
* 오늘의 날씨
* 최신 뉴스

---

### 2. 환각(Hallucination)이 발생한다.

사실이 아닌 내용을 그럴듯하게 생성할 수 있다.

예시)

* 존재하지 않는 논문 인용
* 가짜 법률 조항 생성
* 허위 출처 제공

---

### 3. 외부 시스템을 사용할 수 없다.

LLM 자체는

* 데이터베이스 조회
* 이메일 전송
* 파일 생성
* API 호출

등의 작업을 수행할 수 없다.

---

### 4. 복잡한 작업을 체계적으로 수행하기 어렵다.

단순 질의응답은 잘하지만,

* 정보 검색
* 분석
* 의사결정
* 결과 정리

와 같은 다단계 작업은 한 번의 프롬프트만으로 수행하기 어렵다.

---

# LLM의 한계를 보완하는 기술

## 1. 프롬프트 엔지니어링 (Prompt Engineering)

### 개념

모델의 학습을 변경하지 않고 프롬프트를 최적화하여 원하는 결과를 얻는 방법이다.

### 예시

일반 질문

```text
파이썬을 설명해줘
```

역할 부여

```text
너는 10년차 Python 강사야.
초보자가 이해하기 쉽게 설명해줘.
```

출력 형식 지정

```text
표 형태로 정리해줘.
```

### 장점

* 구현이 쉽다.
* 비용이 적게 든다.
* 즉시 적용 가능하다.

### 단점

* 성능 향상에 한계가 있다.
* 복잡한 도메인 지식을 학습시키기는 어렵다.

---

## 2. 파인튜닝 (Fine-Tuning)

### 개념

기존 LLM을 특정 데이터로 추가 학습시키는 방법이다.

### 예시

* 의료 상담 모델
* 법률 상담 모델
* 기업 내부 업무용 모델

### 장점

* 특정 도메인 성능 향상
* 일관된 답변 생성

### 단점

* 학습 비용이 높다.
* 관련 데이터가 많이 필요하다.
* 최신 정보 반영을 위해 지속적으로 재학습해야 한다.
* 개인 리소스로 운영하기 쉽지 않다.

---

## 3. RAG (Retrieval-Augmented Generation)

### 개념

LLM이 답변하기 전에 외부 문서를 검색하여 참고하도록 만드는 기술이다.

### 동작 과정

1. 사용자 질문 입력
2. 관련 문서 검색
3. 검색 결과를 프롬프트에 추가
4. LLM이 답변 생성

### 예시

사용자 질문

```text
우리 회사 휴가 규정을 알려줘
```

RAG 시스템

```text
사내 규정 문서를 검색
→ 관련 내용 추출
→ LLM에게 전달
```

### 장점

* 최신 정보 활용 가능
* 환각 감소
* 기업 문서 활용 가능

### 단점

* 검색 품질에 의존
* 벡터DB 구축 필요

---

## 4. Function Calling / Tool Calling

### 개념

LLM이 외부 도구를 사용할 수 있도록 하는 기술이다.

### 예시

사용자 요청

```text
오늘 서울 날씨 알려줘
```

LLM

```json
{
  "tool": "get_weather",
  "location": "Seoul"
}
```

도구 실행 후

```text
서울은 현재 27도입니다.
```

### 활용 사례

* 날씨 조회
* 주가 조회

### 장점

* 실시간 데이터 활용
* 실제 업무 수행 가능

### 단점

* Tool 설계 필요
* 보안 고려 필요

### 특징

LLM이 "생각"만 하는 것이 아니라 실제 행동(Action)을 수행하게 만드는 기술이다.

AI Agent의 핵심 요소 중 하나이다.

---

## 5. LangChain

### 개념

LLM 애플리케이션 개발을 위한 프레임워크이다.

### 제공 기능

* Prompt 관리
* Memory
* Tool 연결
* RAG 구현
* Agent 구현

### 예시

```python
prompt → LLM → 출력
```

을

```python
PromptTemplate
→ Retriever
→ LLM
→ OutputParser
```

형태로 구성 가능하다.

### 장점

* 빠른 개발 가능
* 다양한 LLM 지원
* 풍부한 생태계

### 단점

* 추상화가 많음
* 복잡한 Agent 구현 시 한계

### 특징

LangChain은 AI 서비스를 만들기 위한 레고 블록 같은 느낌이다.

필요한 기능을 조립해서 애플리케이션을 만들 수 있다.

---

## 6. LangGraph

### 개념

Agent Workflow를 그래프 구조로 설계하는 프레임워크이다.

LangChain의 한계를 보완하기 위해 등장했다.

### 특징

* 상태(State) 관리
* 조건 분기
* 반복 수행
* 멀티 에이전트

### 예시

```text
질문 분석
    ↓
문서 검색
    ↓
답변 생성
    ↓
검증
    ↓
최종 출력
```

### 장점

* 복잡한 Agent 구현 가능
* 워크플로우 제어 용이
* 멀티 에이전트 지원

### 단점

* 학습 난이도가 높음
* 설계 복잡도 증가

### 특징

LangChain이 함수들을 연결하는 수준이라면,

LangGraph는 여러 Agent가 협업하는 시스템을 설계하는 도구에 가깝다.

최근 AI Agent 개발의 핵심 기술로 자리 잡고 있다.

---

# 정리

LLM의 한계를 보완하기 위해 다양한 기술이 등장하였다.

| 기술                 | 해결하는 문제               |
| ------------------ | --------------------- |
| Prompt Engineering | 출력 품질 향상              |
| Fine-Tuning        | 특정 도메인 성능 향상          |
| RAG                | 최신 정보 활용              |
| Tool Calling       | 외부 시스템 사용             |
| LangChain          | LLM 애플리케이션 개발         |
| LangGraph          | 복잡한 Agent Workflow 구현 |

## 느낀 점

LLM 단독으로는 실제 서비스를 만들기 어렵다.

현재의 AI Agent는 단순히 LLM을 사용하는 것이 아니라 RAG, Tool Calling, LangChain, LangGraph 등의 기술을 조합하여 구축된다는 점을 이해할 수 있었다.

특히 Tool Calling과 LangGraph는 앞으로 AI Agent 개발에서 가장 중요한 기술이 될 것으로 보인다.
