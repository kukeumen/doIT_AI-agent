# 1. PDF 문서 전처리하기

## PyMuPDF란?

PyMuPDF는 PDF 파일의 텍스트, 이미지, 메타데이터 등을 손쉽게 추출할 수 있는 Python 라이브러리이다.

이번 실습에서는 논문의 각 페이지에서 텍스트를 추출하는 작업을 위해 사용하였다.

### 전처리가 필요한 이유

논문 PDF에는 본문 외에도 다음과 같은 정보가 반복적으로 포함된다.

- Header(페이지마다 반복되는 논문 제목)
- Footer(페이지 번호, 저널 정보, 저작권 문구 등)

이러한 내용은 모든 페이지마다 반복되기 때문에 AI가 핵심 내용을 이해하는 데 방해가 될 수 있다.

따라서 반복되는 Header와 Footer를 제거하여 본문만 남기도록 전처리를 수행하였다.

### 전처리 과정

1. PDF 파일 열기
2. 페이지별 텍스트 추출
3. Header 제거
4. Footer 제거
5. 전체 텍스트 합치기
6. txt 파일로 저장

# 2. 논문 요약하기

전처리된 txt 파일을 불러와 LLM에게 논문 요약을 요청하였다.

PDF를 직접 입력하는 대신 txt 파일을 사용하는 이유는

- 불필요한 정보를 제거할 수 있고
- AI가 읽기 쉬운 형태가 되며
- 동일한 데이터를 여러 번 재사용할 수 있기 때문이다.

---

## 출력 형식 지정하기

LLM은 프롬프트에 따라 출력 결과가 달라진다.

단순히

> "논문을 요약해줘."

라고 요청하는 것보다 원하는 출력 형식을 명확하게 제시하면 훨씬 일관된 결과를 얻을 수 있다.

예시 프롬프트

```text
너는 다음 글을 요약하는 봇이다.

아래 글을 읽고,
저자의 문제 인식과 주장을 파악하고,
주요 내용을 요약하라.

작성해야 하는 포맷은 다음과 같다.

# 제목

## 저자의 문제 인식 및 주장
(15문장 이내)

## 저자 소개
```

이처럼 출력 형식을 지정하면 Markdown 형태의 깔끔한 결과를 얻을 수 있으며, 이후 GitHub README나 노트 작성에도 바로 활용할 수 있다.

# 💡 학습하며 알게 된 점

- PDF는 그대로 AI에게 전달하기보다 전처리를 거치는 것이 좋다.
- Header와 Footer 같은 반복 정보는 AI의 이해를 방해할 수 있다.
- PyMuPDF를 활용하면 PDF의 텍스트를 손쉽게 추출할 수 있다.

- 좋은 결과를 얻기 위해서는 프롬프트뿐만 아니라 출력 형식까지 함께 설계하는 것이 중요하다.
