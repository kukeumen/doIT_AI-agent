# 1. GPT에게 역할 부여 (Prompting)

LLM(Large Language Model)은 **System Prompt**를 통해 특정 역할이나 성격을 부여할 수 있다. 역할을 지정하면 동일한 질문이라도 답변의 말투와 관점이 달라진다.

이번 실습에서는 다음과 같은 역할을 부여하였다.

* **배트맨의 조커**

  * 혼란을 즐기고 장난기 있는 말투로 답변
  * 악당의 성격을 유지하며 대화 진행

* **백설공주의 마법 거울**

  * 거울이 왕비에게 이야기하는 듯한 말투 사용
  * 신비롭고 고풍스러운 분위기의 답변 생성

### 핵심 내용

* System Prompt는 AI의 행동 방식을 결정한다.
* 역할을 구체적으로 정의할수록 일관성 있는 답변을 얻을 수 있다.
* 서비스 목적에 따라 상담사, 교사, 면접관 등 다양한 역할을 부여할 수 있다.

# 2. 원하는 답변 유도하기

## One-shot Prompting

One-shot Prompting은 **예시를 하나 제공한 후 원하는 형식의 답변을 생성하도록 하는 방법**이다.

예를 들어,

> 입력 : "사과"
>
> 출력 : "과일입니다."

와 같은 예시를 하나 보여준 뒤 새로운 입력을 제공하면 비슷한 형식으로 답변한다.

### 특징

* 간단하게 원하는 출력 형식을 알려줄 수 있다.
* 출력 형식이 일정해진다.

---

## Few-shot Prompting

Few-shot Prompting은 **여러 개의 예시를 제공하여 패턴을 학습하도록 하는 방법**이다.

예시를 여러 개 보여주면 GPT는 공통된 규칙을 이해하여 더욱 일관된 결과를 생성한다.

### 장점

* 답변의 품질 향상
* 출력 형식의 일관성 유지
* 복잡한 작업에서도 높은 정확도

# 3. 기억하는 챗봇 만들기

## Single-turn Chatbot

Single-turn 챗봇은 **현재 질문만을 기반으로 답변**한다.

```text
사용자 : 내 이름은 교동이야.
GPT : 반가워!

사용자 : 내 이름이 뭐야?
GPT : 알 수 없습니다.
```

이전 대화를 저장하지 않기 때문에 문맥(Context)이 유지되지 않는다.

---

## Multi-turn Chatbot

Multi-turn 챗봇은 이전 대화 내역(Messages)을 함께 전달하여 **대화의 흐름을 기억**한다.

```text
사용자 : 내 이름은 배츠마루야.
GPT : 반가워!

사용자 : 내 이름이 뭐야?
GPT : 배츠마루입니다.
```

### 핵심 원리

OpenAI API는 대화를 자동으로 기억하지 않는다.

따라서 매 요청마다 이전 대화를 `messages` 리스트에 저장하여 함께 전달해야 한다.

```python
messages.append({"role":"user","content":user_input})
messages.append({"role":"assistant","content":response})
```

실제 AI 서비스에서는 이러한 대화 기록을 활용하여 자연스러운 챗봇을 구현한다.

# 4. Streamlit으로 웹 챗봇 구현

지금까지는 터미널에서 챗봇을 실행하였다.

이번 실습에서는 **Streamlit**을 이용하여 웹 브라우저에서 사용할 수 있는 챗봇을 구현하였다.

Streamlit은 Python만으로도 웹 UI를 쉽게 만들 수 있는 프레임워크이다.

주요 기능

* `st.chat_input()` : 사용자 입력
* `st.chat_message()` : 채팅 UI 출력
* `st.session_state` : 대화 기록 저장

특히 `session_state`를 이용하면 새로고침 전까지 대화 내용을 유지할 수 있어 Multi-turn 챗봇 구현에 매우 적합하다.

이번 실습에서는 단순히 GPT를 호출하는 것을 넘어 **프롬프트 과정을 설계하는 방법**을 학습하였다. 역할 부여(Role Prompting)와 One-shot, Few-shot Prompting을 통해 원하는 형태의 답변을 유도할 수 있었으며, Single-turn과 Multi-turn의 차이를 이해하면서 대화 기록 관리의 중요성을 확인하였다. 또한 Streamlit을 활용하여 웹 기반 챗봇을 구현함으로써 실제 AI 서비스를 개발하는 기본적인 구조를 익힐 수 있었다.
